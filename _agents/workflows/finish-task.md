# 작업 완료 보고 워크플로우 (Finish Task Hook)

단일 작업이나 마일스톤이 완전히 마무리되었을 때, 단순하게 "다 했습니다"라고 보고하지 않고 아래 형식을 갖추어 제출합니다.

1. **보고서 문서 갱신 (walkthrough.md)**
   - **무엇을 발견했는가:** 기존 시스템 구조에서 발견한 특이점이나 문제점
   - **무엇을 수정했는가:** 실제로 코드를 변경한 부분과 로직의 차이
   - **왜 그렇게 판단했는가 (의사결정 근거):** 대안이 있었음에도 현재 방식을 선택한 이유

2. **AI 교차 검증 (Cross-Review 모드 장착)**
   - 스스로를 '엄격한 시니어 코드 리뷰어' 페르소나에 맞춰 빙의합니다.
   - 작성한 코드 중 아쉬운 곳, 누락된 곳, 테스트가 아직 미비한 곳이 있는지 다시 한번 점검하고 이를 보고서 말미에 "한계점 및 향후 개선사항(Next Steps)"으로 명시합니다.

3. **최종 사용자 보고**
   - 갱신된 `walkthrough.md` 를 첨부하여 사용자에게 최종 보고(`notify_user`)합니다.
