# 자동 품질 검사 및 리뷰 (Quality Check Hook)

코드의 특정 단계 구현이 끝나면 무조건 실행해야 하는 절차입니다. 불량 코드를 넘기지 않기 위한 자체 검수 라인입니다.

1. **수정된 파일 확인 (CCTV 로깅)**
   - 어떤 파일을 수정했는지 정확히 파악하고 나열합니다.

2. **자동 검증 실행**
   - `python -m pytest tests/unit/ -q` (또는 해당하는 단위 테스트)를 실행합니다.
   - `python -m ruff check .` 와 `python -m mypy .` (또는 ESLint 등 환경에 맞는 Lint/Type 체커)를 실행합니다.
   - 위 명령어가 실패하면(에러가 나오면) 자동으로 버그를 분석하고 수정 작업을 진행합니다.

3. **Self-Correction 넛지 질문**
   코드에 문제가 없다고 판단되더라도 스스로 아래 질문을 던지고 답변을 확인합니다:
   - "오류 처리(Exception Handling, `try-except`)는 적절히 되어 있는가?"
   - "보안상 위험한 부분이나 자원(DB, 파일) 누수 발생 가능성은 없는가?"
   - "기존 동작(Regression)을 망가뜨리는 충돌 코드는 없는가?"

4. **검사 완료 보고**
   - 검사가 통과했거나, 문제점을 발견해 수정했다면 그 상태를 사용자와 공유합니다.
