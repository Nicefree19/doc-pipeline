# 작업 시작 워크플로우 (Start Task Hook)

새로운 기능을 구현하거나 버그를 픽스하기 전에, 코딩을 바로 시작하지 않고 아래 단계를 순차적으로 거칩니다.

1. **상황 파악 및 가이드 열람**
   - 사용자 요청 키워드(예: `backend`, `frontend`, `test`, `.py` 등)를 확인합니다.
   - 프로젝트 내 `guides/ai_agent_sop.md` 및 관련된 세부 가이드 문서를 반드시 `view_file` 로 열람하여 컨텍스트에 포함시킵니다.

2. **작업 기억 장치 (문서) 생성 및 업데이트**
   - **`implementation_plan.md` 생성/갱신:** 무엇을 어떻게 만들 것인지, 아키텍처와 흐름, 그리고 왜 이 기술을 선택했는지(맥락 노트) 기록합니다.
   - **`task.md` 생성/갱신 (체크리스트):** 완료할 작업과 남은 업무를 잘게 쪼개어 추적 가능하도록 만듭니다.

3. **승인 대기 (중요 - 여기서 멈춤)**
   - 위 2번의 문서가 작성 완료되면 즉시 작업을 멈추고 `notify_user` 를 호출하여 
     **"계획서와 체크리스트를 업데이트했습니다. 코딩을 시작하기 전에 확인해 주시면 다음 단계(1단계 구현)를 진행하겠습니다."** 라고 보고합니다.
