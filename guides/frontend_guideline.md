# 프론트엔드 개발 표준 지침 (Frontend Guideline)

이 문서에는 UI 요소 변경, 컴포넌트 개발 시 알아야 할 디자인 규칙과 상태 관리 패턴을 정의합니다.

## 1. 상태 관리 (State Management)
- 지역 상태(Local state)와 전역 상태(Global state)를 명확히 구분합니다.
- 비동기 데이터 처리(API 호출 등)에는 Loading, Error, Success 상태를 명확하게 구분하여 사용자가 인지할 수 있도록 UI에 반영합니다.

## 2. 컴포넌트 설계 및 분리
- 하나의 컴포넌트에 여러 역할이 섞이지 않도록 **단일 책임 원칙**을 지킵니다.
- 프레젠테이셔널 컴포넌트(UI 전용)와 컨테이너 컴포넌트(로직 및 데이터 fetch 전용)로 구분하는 구조적 접근을 유지합니다.
- Magic Number나 하드코딩된 문자열 대신 상수(Constants) 파일이나 테마/설정 파일에서 값을 가져와 사용합니다.

## 3. 접근성 및 UX 방어 코딩
- 사용자가 버튼을 여러 번 누르지 못하도록 디바운싱(Debouncing) 혹은 비활성화(Disable) 처리를 적용합니다.
- 입력 폼에서는 에러 처리를 즉각적으로 유효성 검사해 알려줍니다.

## 4. 반응형 및 스타일링
- 다양한 화면비(16:9 뷰, 모바일 뷰)를 고려하여 미디어 쿼리 혹은 레이아웃 프레임워크 기능(Tailwind 등)을 적용합니다.
