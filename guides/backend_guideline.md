# 백엔드 개발 표준 지침 (Backend Guideline)

이 지침은 Python, Node.js 또는 Java 등 백엔드 환경에서 기능을 추가하거나 리팩토링할 때 AI 에이전트가 기본적으로 고려해야 할 아키텍처 및 품질 기준입니다.

## 1. 아키텍처 및 구조화
- 비즈니스 로직과 라우팅 로직을 철저히 분리합니다.
- 데이터베이스 접근 계층(Repository 또는 DAO)과 서비스를 분리합니다.
- 상태(State) 추적이 가능하도록 핵심 변경점에는 적절한 로거(`logger.info` 등)를 추가합니다.

## 2. 보안 및 오류 처리 (Self-Review 필수)
- **예외 처리 (Exception Handling):** 모든 외부 입출력(네트워크, 디스크 등)에는 `try-except` 블록을 포함하여 방어적으로 코딩합니다.
- 예외 발생 시 에러 메세지를 억제(swallow)하지 말고, 반드시 적절한 레벨(`logger.error` 또는 `logger.exception`)로 기록합니다.
- 입력값에 대한 검증(Validation)을 통과한 데이터만 처리합니다.

## 3. 리소스 및 자원 누수 관리
- 열린 파일, 데이터베이스 커넥션, 스레드는 작업 완료 후 반드시 명시적으로 정리(Close, Shutdown 등)합니다.
- 컨텍스트 매니저(`with` 구문) 사용을 우선합니다.

## 4. 검증을 위한 상태
- 백엔드 작업 시, 해당 코드가 어떤 상황에서 동작하는지 식별하기 쉽도록 식별자/트래킹 ID 부여 등 명확성을 유지합니다.
